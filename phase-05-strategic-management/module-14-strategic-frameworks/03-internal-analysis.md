# Internal Analysis Frameworks

## Learning Objectives

- Apply the VRIO framework to assess resource-based advantages
- Use the McKinsey 7S model for organizational analysis
- Identify and evaluate core competencies
- Conduct value chain analysis to find competitive advantage
- Assess organizational capabilities systematically

---

## Introduction

External analysis tells us where to compete; internal analysis tells us how well-equipped we are to compete. Understanding your organization's strengths and weaknesses—its resources, capabilities, and organizational alignment—is essential for strategy formulation. This chapter covers the key frameworks for internal strategic analysis.

---

## VRIO Framework

### Resource-Based View

```
RESOURCE-BASED VIEW (RBV):
─────────────────────────────────────────────────────
Competitive advantage comes from valuable, rare,
inimitable resources and capabilities

RESOURCE TYPES:
─────────────────────────────────────────────────────
Tangible Resources:
• Financial (cash, borrowing capacity)
• Physical (facilities, equipment, location)
• Technological (patents, R&D facilities)
• Organizational (planning systems, databases)

Intangible Resources:
• Human (skills, knowledge, relationships)
• Innovation (new product capability)
• Reputation (brand, customer trust)
• Culture (values, norms, beliefs)

CAPABILITIES:
─────────────────────────────────────────────────────
Ability to deploy resources effectively
Often combinations of resources
Developed through experience
Embedded in routines
```

### VRIO Analysis

```
VRIO FRAMEWORK:
─────────────────────────────────────────────────────

V - VALUABLE?
    Does it enable the firm to exploit opportunities
    or neutralize threats?

    If NO → Competitive DISADVANTAGE
    If YES → Continue...

R - RARE?
    Is it controlled by only a small number of
    competing firms?

    If NO → Competitive PARITY
    If YES → Continue...

I - COSTLY TO IMITATE?
    Do firms without it face cost disadvantage
    in obtaining it?

    If NO → TEMPORARY ADVANTAGE
    If YES → Continue...

O - ORGANIZED TO CAPTURE VALUE?
    Is the firm organized to exploit the resource?

    If NO → Unused POTENTIAL
    If YES → SUSTAINED COMPETITIVE ADVANTAGE

VRIO DECISION TREE:
─────────────────────────────────────────────────────
                    Is it Valuable?
                         │
             ┌───────────┼───────────┐
             NO                      YES
             │                       │
    Competitive              Is it Rare?
    Disadvantage                     │
                         ┌───────────┼───────────┐
                         NO                      YES
                         │                       │
                  Competitive           Costly to Imitate?
                    Parity                       │
                                     ┌───────────┼───────────┐
                                     NO                      YES
                                     │                       │
                              Temporary          Organized to Capture?
                              Advantage                      │
                                             ┌───────────────┼───────────┐
                                             NO                          YES
                                             │                           │
                                      Unused                     Sustained
                                      Potential                  Advantage
```

### VRIO Application Example

```
VRIO ANALYSIS: Apple Inc.
─────────────────────────────────────────────────────

Resource/Capability    V    R    I    O    Implication
─────────────────────────────────────────────────────
Brand reputation       Y    Y    Y    Y    Sustained advantage
Ecosystem integration  Y    Y    Y    Y    Sustained advantage
Design capability      Y    Y    Y    Y    Sustained advantage
Retail stores          Y    Y    N    Y    Temporary advantage
Cash reserves          Y    N    N    Y    Competitive parity
Manufacturing scale    Y    N    N    Y    Competitive parity
Supply chain           Y    Y    N    Y    Temporary advantage

WHY IMITATION IS COSTLY:
─────────────────────────────────────────────────────
Brand: Built over decades, emotional connection
Ecosystem: Network effects, switching costs, complexity
Design: Unique culture, talent, leadership

STRATEGIC IMPLICATIONS:
─────────────────────────────────────────────────────
• Invest in protecting/extending brand
• Deepen ecosystem integration
• Maintain design excellence
• Supply chain can be imitated—not source of advantage
```

## McKinsey 7S Framework

### Framework Overview

```
McKINSEY 7S MODEL:
─────────────────────────────────────────────────────
Seven interconnected elements for organizational
effectiveness—all must be aligned

            ┌───────────────┐
            │   STRATEGY    │
            └───────┬───────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐     ┌─────▼─────┐     ┌───▼───┐
│STRUCTURE│◄──►│  SHARED   │◄──►│SYSTEMS│
└───┬───┘     │  VALUES   │     └───┬───┘
    │         └─────┬─────┘         │
    │               │               │
    └───────────────┼───────────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐     ┌─────▼─────┐     ┌───▼───┐
│ STYLE │◄──►│   STAFF   │◄──►│SKILLS │
└───────┘     └───────────┘     └───────┘

HARD S's (Easier to change):
• Strategy, Structure, Systems

SOFT S's (Harder to change):
• Style, Staff, Skills, Shared Values
```

### 7S Elements Detail

```
STRATEGY:
─────────────────────────────────────────────────────
Plan for building competitive advantage
• What is the strategy?
• Is it clear and understood?
• Does it create advantage?

STRUCTURE:
─────────────────────────────────────────────────────
Organization design and reporting relationships
• How is work divided?
• Who reports to whom?
• How centralized/decentralized?

SYSTEMS:
─────────────────────────────────────────────────────
Processes and procedures for work
• How is information shared?
• What are the key processes?
• How is performance measured?

SHARED VALUES:
─────────────────────────────────────────────────────
Core beliefs and culture
• What does the organization believe?
• What behaviors are valued?
• What brings people together?

STYLE:
─────────────────────────────────────────────────────
Leadership approach and management style
• How do leaders behave?
• How are decisions made?
• How is conflict handled?

STAFF:
─────────────────────────────────────────────────────
People and talent management
• What talent exists?
• How are people developed?
• What are talent gaps?

SKILLS:
─────────────────────────────────────────────────────
Organizational capabilities
• What is the organization good at?
• What distinctive competencies?
• What capabilities are missing?
```

### 7S Application

```
7S ANALYSIS: Struggling Retail Company
─────────────────────────────────────────────────────

Element        Current State        Issue
─────────────────────────────────────────────────────
Strategy       Omnichannel focus    Clear but not executed
Structure      Siloed (online vs    Conflict between channels
               stores separate)
Systems        Legacy IT systems    Can't support omnichannel
Shared Values  "Stores first"       Culture resists change
Style          Command and control  Slow decision-making
Staff          Retail background    Lacking digital talent
Skills         Merchandising        Weak in digital/analytics

MISALIGNMENTS:
─────────────────────────────────────────────────────
Strategy ↔ Structure: Strategy is omnichannel but
                      structure keeps channels separate

Strategy ↔ Skills: Strategy requires digital skills
                   organization doesn't have

Shared Values ↔ Strategy: Culture values stores,
                          strategy requires digital

RECOMMENDATIONS:
─────────────────────────────────────────────────────
1. Restructure around customer, not channel
2. Invest in new technology systems
3. Hire digital talent (Staff)
4. Change values through leadership (Style)
5. Build new skills through training
```

## Core Competencies

### Identifying Core Competencies

```
CORE COMPETENCY DEFINITION (Hamel & Prahalad):
─────────────────────────────────────────────────────
A bundle of skills and technologies that:
1. Provides access to wide variety of markets
2. Makes significant contribution to customer benefits
3. Is difficult for competitors to imitate

NOT Core Competencies:
─────────────────────────────────────────────────────
• Things you do well but aren't distinctive
• Single skills or technologies
• Products (competencies underlie products)

COMPETENCY HIERARCHY:
─────────────────────────────────────────────────────
               Products/Services
                      │
                      ▼
              Core Products
                      │
                      ▼
              Core Competencies
                      │
                      ▼
         Skills + Technologies + Routines
```

### Core Competency Examples

```
CLASSIC EXAMPLES:
─────────────────────────────────────────────────────

HONDA:
Core competency: Engine design and manufacturing
Applies to: Motorcycles, cars, lawnmowers, generators
Customer benefit: Reliability, fuel efficiency
Difficult to imitate: Decades of learning

SONY (historical):
Core competency: Miniaturization
Applies to: Walkman, cameras, TVs, phones
Customer benefit: Portability
Difficult to imitate: Technical expertise

AMAZON:
Core competency: Operational logistics at scale
Applies to: Retail, AWS, streaming, devices
Customer benefit: Speed, convenience, reliability
Difficult to imitate: Infrastructure investment

APPLE:
Core competency: Integrated design (HW+SW+Services)
Applies to: Mac, iPhone, iPad, Watch, Services
Customer benefit: User experience
Difficult to imitate: Culture, talent, history
```

```
CORE COMPETENCY AUDIT:
─────────────────────────────────────────────────────

ASSESSMENT TEMPLATE:
```

## Capability Assessment

### Capability Mapping

```
CAPABILITY ASSESSMENT FRAMEWORK:
─────────────────────────────────────────────────────

CAPABILITY CATEGORIES:
─────────────────────────────────────────────────────
Innovation: R&D, new product development
Operations: Manufacturing, supply chain, quality
Marketing: Brand, customer insight, digital
Sales: Relationship management, channel management
Service: Customer support, after-sales
Technology: IT, data analytics, digital platforms
Organization: Talent, culture, execution

ASSESSMENT DIMENSIONS:
─────────────────────────────────────────────────────
• Current level (1-5 scale)
• Importance to strategy (1-5 scale)
• Trend (improving/declining/stable)
• Gap vs. competitors
• Investment priority

CAPABILITY MATRIX:
─────────────────────────────────────────────────────
                    CURRENT CAPABILITY
                    Weak            Strong
              ┌─────────────────┬─────────────────┐
         High │   BUILD         │   LEVERAGE      │
  STRATEGIC   │   (Priority     │   (Exploit      │
  IMPORTANCE  │   investment)   │   fully)        │
              ├─────────────────┼─────────────────┤
         Low  │   CONSIDER      │   MAINTAIN      │
              │   (If cheap)    │   (Don't over-  │
              │                 │   invest)       │
              └─────────────────┴─────────────────┘
```

### Strengths and Weaknesses Summary

```
INTERNAL ANALYSIS SYNTHESIS:
─────────────────────────────────────────────────────

STRENGTHS (leverage these):
─────────────────────────────────────────────────────
From VRIO: Resources with sustained advantage
From 7S: Aligned elements
From competencies: Core competencies identified
From capabilities: Strong capabilities

WEAKNESSES (address these):
─────────────────────────────────────────────────────
From VRIO: Resources creating disadvantage
From 7S: Misaligned elements
From competencies: Missing competencies
From capabilities: Weak capabilities

PRIORITIZATION:
─────────────────────────────────────────────────────
High priority: Weaknesses that undermine strategy
               Strengths that differentiate
Medium: Other significant items
Low: Not material to strategic success
```

## Key Takeaways

1. **VRIO identifies sustainable advantages** - Valuable, rare, inimitable, organized
2. **7S reveals organizational alignment** - All elements must work together
3. **Core competencies underlie products** - Skills and technologies that differentiate
4. **Capabilities can be assessed systematically** - Current vs. required
5. **Internal and external analysis connect** - Match strengths to opportunities
6. **Weaknesses may block strategy** - Must be addressed or worked around
7. **Change starts with diagnosis** - Understand before prescribing
8. **Soft factors are harder** - Culture and skills change slowly
