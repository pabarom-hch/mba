# Framework Integration

## Learning Objectives

- Combine multiple frameworks for comprehensive analysis
- Select appropriate frameworks for specific situations
- Develop custom frameworks when standard tools don't fit
- Avoid common framework pitfalls
- Apply integrated analysis to complex strategic problems

---

## Introduction

Real strategic problems rarely fit neatly into a single framework. Effective strategists combine multiple frameworks, adapt standard tools, and sometimes create new ones. This chapter teaches how to integrate frameworks into coherent strategic analysis and when to develop custom approaches.

---

## Choosing Frameworks

### Framework Selection Guide

```
MATCHING FRAMEWORKS TO QUESTIONS:
─────────────────────────────────────────────────────

QUESTION TYPE              FRAMEWORKS TO USE
─────────────────────────────────────────────────────

"Is this industry attractive?"
→ Five Forces, PESTEL, Industry Lifecycle

"What's our competitive position?"
→ VRIO, Value Chain, Strategic Group Mapping

"How should we compete?"
→ Generic Strategies, Blue Ocean, Positioning

"Where should we grow?"
→ Ansoff Matrix, Market Attractiveness

"What businesses should we be in?"
→ BCG Matrix, GE-McKinsey, Parenting Matrix

"Is our organization aligned?"
→ 7S Model, Capability Assessment

"How will competitors respond?"
→ Game Theory, Competitor Analysis

"How do we implement?"
→ 100-Day Plan, Balanced Scorecard, Kotter
```

### Framework Combinations

```
COMMON FRAMEWORK COMBINATIONS:
─────────────────────────────────────────────────────

STRATEGIC ASSESSMENT:
External: PESTEL + Five Forces + Competitor Analysis
Internal: VRIO + 7S + Value Chain
Synthesis: SWOT → Strategic Options

COMPETITIVE STRATEGY:
Industry: Five Forces + Strategic Groups
Position: Generic Strategies + Value Curve
Dynamic: Game Theory + First-Mover Analysis

CORPORATE STRATEGY:
Portfolio: BCG Matrix + GE-McKinsey
Parenting: 7S + Parenting Matrix
Growth: Ansoff + M&A Framework

IMPLEMENTATION:
Planning: 100-Day Plan + Initiative Prioritization
Change: Kotter + Stakeholder Analysis
Monitoring: Balanced Scorecard + Strategic Reviews
```

## Integrated Analysis Process

### Step-by-Step Integration

```
COMPREHENSIVE STRATEGIC ANALYSIS:
─────────────────────────────────────────────────────

PHASE 1: EXTERNAL ANALYSIS
─────────────────────────────────────────────────────
Step 1: PESTEL scan (macro trends)
Step 2: Five Forces analysis (industry structure)
Step 3: Industry lifecycle (market maturity)
Step 4: Competitor analysis (key rivals)
Step 5: Strategic group mapping (competitive clusters)

Output: Opportunities and threats identified
        Industry attractiveness assessed

PHASE 2: INTERNAL ANALYSIS
─────────────────────────────────────────────────────
Step 1: Value chain analysis (activities)
Step 2: VRIO assessment (resources/capabilities)
Step 3: Core competency identification
Step 4: 7S alignment check (organization)
Step 5: Financial analysis (performance)

Output: Strengths and weaknesses identified
        Competitive advantage assessed

PHASE 3: STRATEGIC OPTIONS
─────────────────────────────────────────────────────
Step 1: SWOT synthesis (combine external/internal)
Step 2: TOWS matrix (generate options)
Step 3: Ansoff matrix (growth options)
Step 4: Blue Ocean (differentiation options)
Step 5: Game theory (competitive dynamics)

Output: Strategic options articulated
        Trade-offs understood

PHASE 4: STRATEGY SELECTION
─────────────────────────────────────────────────────
Step 1: Evaluate options against criteria
Step 2: Assess feasibility and risk
Step 3: Test strategic fit
Step 4: Financial modeling
Step 5: Select and commit

Output: Chosen strategy with rationale

PHASE 5: IMPLEMENTATION
─────────────────────────────────────────────────────
Step 1: 100-day planning
Step 2: Kotter change management
Step 3: Balanced Scorecard metrics
Step 4: Governance structure
Step 5: Communication plan

Output: Actionable implementation plan
```

## Custom Framework Development

### When to Create Custom Frameworks

```
STANDARD FRAMEWORKS INSUFFICIENT WHEN:
─────────────────────────────────────────────────────
• Problem doesn't fit standard categories
• Industry is unique (emerging, disrupted)
• Question is highly specific
• Standard framework overlooks key factors
• Need simpler or more complex analysis

CUSTOM FRAMEWORK PRINCIPLES:
─────────────────────────────────────────────────────
1. MECE (mutually exclusive, collectively exhaustive)
2. Issue-focused (addresses the question)
3. Actionable (leads to decisions)
4. Simple as possible (but not simpler)
5. Visual when helpful
```

### Creating Custom Frameworks

```
CUSTOM FRAMEWORK DEVELOPMENT:
─────────────────────────────────────────────────────

STEP 1: DEFINE THE QUESTION
─────────────────────────────────────────────────────
What specific question needs answering?
What decision will result?

STEP 2: IDENTIFY KEY FACTORS
─────────────────────────────────────────────────────
What factors drive the answer?
Brainstorm, then prioritize

STEP 3: ORGANIZE MECE
─────────────────────────────────────────────────────
Group factors logically
Ensure complete coverage
No overlaps

STEP 4: STRUCTURE VISUALLY
─────────────────────────────────────────────────────
Matrix, tree, process, or other format
Make intuitive to use

STEP 5: TEST AND REFINE
─────────────────────────────────────────────────────
Apply to real example
Refine based on results

EXAMPLE: DIGITAL DISRUPTION READINESS
─────────────────────────────────────────────────────

                INDUSTRY VULNERABILITY
                Low               High
          ┌─────────────────┬─────────────────┐
     High │   OPPORTUNITY   │   URGENT        │
INTERNAL  │   SEEKER        │   TRANSFORMER   │
DIGITAL   │                 │                 │
CAPABILITY│   Pursue digital│   Transform now │
          │   opportunities │   or perish     │
          ├─────────────────┼─────────────────┤
     Low  │   MONITOR       │   VULNERABLE    │
          │                 │                 │
          │   Watch and     │   Build         │
          │   prepare       │   capability    │
          └─────────────────┴─────────────────┘
```

## Common Pitfalls

### Framework Misuse

```
COMMON FRAMEWORK MISTAKES:
─────────────────────────────────────────────────────

1. FRAMEWORK WORSHIP
   Mistake: Forcing data into framework
   Fix: Start with question, not framework

2. ANALYSIS PARALYSIS
   Mistake: Too many frameworks, no decision
   Fix: 80/20—use minimum needed

3. STATIC APPLICATION
   Mistake: Snapshot view, ignore dynamics
   Fix: Consider trends and evolution

4. CONFIRMATION BIAS
   Mistake: Framework supports predetermined answer
   Fix: Actively seek disconfirming evidence

5. OVERSIMPLIFICATION
   Mistake: Complex reality forced into 2x2
   Fix: Acknowledge limitations, add nuance

6. INCOMPLETE APPLICATION
   Mistake: Cherry-pick favorable elements
   Fix: Complete the full analysis

7. NO SO-WHAT
   Mistake: Analysis without implications
   Fix: Every analysis must have "so what"
```

### Quality Checklist

```
FRAMEWORK ANALYSIS QUALITY CHECK:
─────────────────────────────────────────────────────

□ Right framework(s) for the question?
□ Complete analysis (all elements covered)?
□ Based on evidence (not assumptions)?
□ Dynamic view (trends, not just snapshot)?
□ Connects to action (clear implications)?
□ Addresses counterarguments?
□ Limitations acknowledged?
□ Synthesized across frameworks?
```

## Synthesis and Communication

### Creating the Narrative

```
FROM ANALYSIS TO STORY:
─────────────────────────────────────────────────────

STEP 1: HEADLINE
What is the single most important message?
"Our competitive advantage is eroding and we must
act now to reposition."

STEP 2: SUPPORTING ARGUMENTS
What are the 3-4 key supporting points?
• Industry becoming more competitive (Five Forces)
• Our advantages being imitated (VRIO)
• Growth opportunities exist in adjacent markets (Ansoff)
• Organization needs realignment (7S)

STEP 3: EVIDENCE
What data supports each point?
Specific findings from each framework

STEP 4: IMPLICATIONS
What should we do?
Strategic recommendations with actions

STEP 5: NEXT STEPS
What happens now?
Concrete immediate actions
```

### Executive Presentation Structure

```
STRATEGY PRESENTATION OUTLINE:
─────────────────────────────────────────────────────

1. EXECUTIVE SUMMARY (1 page/slide)
   • Situation, complication, resolution
   • Key recommendation
   • Expected outcomes

2. CONTEXT (2-3 pages/slides)
   • Market and industry overview
   • Company situation
   • Why action needed now

3. ANALYSIS (5-10 pages/slides)
   • External analysis findings
   • Internal analysis findings
   • Integrated SWOT

4. OPTIONS (3-5 pages/slides)
   • Strategic options considered
   • Evaluation criteria
   • Trade-offs

5. RECOMMENDATION (3-5 pages/slides)
   • Chosen strategy
   • Key initiatives
   • Expected outcomes
   • Investment required

6. IMPLEMENTATION (3-5 pages/slides)
   • 100-day plan
   • Governance
   • Metrics
   • Risks and mitigation

7. APPENDIX (as needed)
   • Detailed analyses
   • Data sources
   • Assumptions
```

## Key Takeaways

1. **Match framework to question** - Right tool for right problem
2. **Combine frameworks thoughtfully** - Integrated analysis
3. **Custom frameworks when needed** - Standard tools have limits
4. **Avoid common pitfalls** - Framework worship, paralysis, bias
5. **Quality over quantity** - Better one good analysis than five superficial
6. **Synthesis is key** - Connect analyses to coherent story
7. **Action orientation** - Every analysis must lead to "so what"
8. **Communicate clearly** - Pyramid structure, answer first
