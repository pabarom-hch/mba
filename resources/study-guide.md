## Case Study Approach

### The Case Method

```
HOW TO ANALYZE A CASE:
─────────────────────────────────────────────────────

STEP 1: QUICK READ (10 min)
─────────────────────────────────────────────────────
• Skim to understand situation
• Note key characters/companies
• Identify time period and context

STEP 2: CAREFUL READ (20 min)
─────────────────────────────────────────────────────
• Read thoroughly
• Mark key facts and figures
• Note exhibits and data

STEP 3: DEFINE THE PROBLEM (10 min)
─────────────────────────────────────────────────────
• What decision must be made?
• What are the constraints?
• Who are the stakeholders?

STEP 4: ANALYZE (30 min)
─────────────────────────────────────────────────────
• Apply relevant frameworks
• Calculate key metrics
• Evaluate alternatives

STEP 5: RECOMMEND (10 min)
─────────────────────────────────────────────────────
• State clear recommendation
• Justify with analysis
• Address implementation
• Note risks and contingencies
```

### Case Analysis Framework

```
STRUCTURED CASE ANALYSIS:
─────────────────────────────────────────────────────

1. SITUATION
   • What is the context?
   • Who is the decision maker?
   • What triggered the need for action?

2. PROBLEM/OPPORTUNITY
   • What is the core issue?
   • What are the sub-issues?
   • What are the constraints?

3. ANALYSIS
   • What frameworks apply?
   • What does the data show?
   • What are the options?

4. RECOMMENDATION
   • What should be done?
   • Why this option?
   • How to implement?

5. RISK/CONTINGENCY
   • What could go wrong?
   • How to mitigate?
   • What's the backup plan?
```

## Assessment Strategy

### Preparing for Exams

```
EXAM PREPARATION CHECKLIST:
─────────────────────────────────────────────────────

□ Review all learning objectives
□ Know key frameworks by heart
□ Practice calculations (finance, ops)
□ Review case study patterns
□ Take practice assessments
□ Focus on weak areas
□ Get adequate rest before exam
```

### Common Exam Formats

```
ASSESSMENT TYPES:
─────────────────────────────────────────────────────

MULTIPLE CHOICE:
• Test concept understanding
• Typically 2-3 points each
• Process of elimination helps

SHORT ANSWER:
• Apply frameworks concisely
• Structure with bullet points
• Show your thinking

CALCULATIONS:
• Show all work
• Label formulas used
• Check reasonableness

CASE ANALYSIS:
• Use structured approach
• Support with data
• Make clear recommendations
```

## Integration Tips

### Building Connections

```
CROSS-FUNCTIONAL THINKING:
─────────────────────────────────────────────────────

As you progress, actively connect concepts:

STRATEGY + FINANCE:
• Strategic decisions → Financial impact
• Valuation → Strategic options

MARKETING + OPERATIONS:
• Customer promise → Operational capability
• Brand → Service delivery

FINANCE + OPERATIONS:
• Working capital → Supply chain
• Capital investment → Capacity

LEADERSHIP + ALL FUNCTIONS:
• Every initiative requires → Change management
• Every team needs → Leadership
```

```
THINKING LIKE A CEO:
─────────────────────────────────────────────────────

Avoid:                   Practice:
─────────────────────────────────────────────────────
Single-function view     Multi-stakeholder view
Short-term focus         Long-term + short-term
```
