
```
┌─────────────────────────────────────────────────────────────────────────────┐

│                                                                             │
```

### Effective Feedback Framework

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    SBI FEEDBACK MODEL                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   S - SITUATION                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   Describe the specific context                                     │   │
│   │   When and where did this occur?                                    │   │
│   │                                                                     │   │
│   │   Example: "In yesterday's client meeting..."                       │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   B - BEHAVIOR                                                              │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   Describe the observable behavior (not interpretation)             │   │
│   │   What did you see or hear?                                         │   │
│   │                                                                     │   │
│   │   ✗ "You were rude"                                                 │   │
│   │   ✓ "You interrupted the client three times"                        │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   I - IMPACT                                                                │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   Explain the effect of the behavior                                │   │
│   │   What was the consequence?                                         │   │
│   │                                                                     │   │
│   │   Example: "...which made the client appear frustrated and she      │   │
│   │   cut the meeting short"                                            │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────────     │
│                                                                             │
│   COMPLETE EXAMPLE:                                                         │
│                                                                             │
│   REINFORCING:                                                              │
│   "In yesterday's client meeting (S), you asked clarifying questions        │
│   and summarized their needs before proposing a solution (B). This          │
│   made the client feel heard and they agreed to move forward (I)."          │
│                                                                             │
│   REDIRECTING:                                                              │
│   "In yesterday's client meeting (S), you interrupted the client            │
│   three times while she was explaining her concerns (B). This               │
│   seemed to frustrate her and she ended the meeting early (I).              │
│   Next time, let's try pausing longer before responding."                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Common Rating Scales

| Type | Scale | Description |
|------|-------|-------------|
| **5-Point** | 1-5 | Unsatisfactory to Exceptional |
| **4-Point** | 1-4 | Forces differentiation (no middle) |
| **3-Point** | Does Not Meet / Meets / Exceeds | Simple, clear categories |
| **No Ratings** | Narrative only | Focus on development, not labels |

### Calibration Process

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    CALIBRATION SESSION                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   PURPOSE: Ensure consistency and fairness across managers                  │
│                                                                             │
│   PARTICIPANTS: Managers + HR + Skip-level leaders                          │
│                                                                             │
│   PROCESS:                                                                  │
│                                                                             │
│   1. PREPARE                                                                │
│      ├── Managers complete preliminary ratings                              │
│      ├── HR compiles distribution data                                      │
│      └── Identify outliers and edge cases                                   │
│                                                                             │
│   2. DISCUSS                                                                │
│      ├── Review rating definitions and standards                            │
│      ├── Managers present rationale for ratings                             │
│      ├── Peers challenge and question                                       │
│      └── Focus on evidence, not opinions                                    │
│                                                                             │
│   3. ADJUST                                                                 │
│      ├── Move ratings based on discussion                                   │
│      ├── Ensure comparable standards                                        │
│      └── Document decisions                                                 │
│                                                                             │
│   4. FINALIZE                                                               │
│      ├── Confirm final distribution                                         │
│      ├── Identify development needs                                         │
│      └── Plan talent actions (promotions, PIPs, etc.)                       │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────────     │
│                                                                             │
│   FORCED DISTRIBUTION vs. GUIDED DISTRIBUTION                               │
│                                                                             │
│   FORCED:                          GUIDED:                                  │
│   ┌─────────────────────┐         ┌─────────────────────┐                   │
│   │ Exceptional: 10%    │         │ Exceptional: 10-15% │                   │
│   │ Exceeds: 20%        │         │ Exceeds: 20-30%     │                   │
│   │ Meets: 50%          │         │ Meets: 40-60%       │                   │
│   │ Below: 15%          │         │ Below: 10-15%       │                   │
│   │ Unsatisfactory: 5%  │         │ Unsatisfactory: <5% │                   │
│   └─────────────────────┘         └─────────────────────┘                   │
│                                                                             │
│   Most companies moving away from rigid forced distribution                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Addressing Performance Issues

### Performance Improvement Plan (PIP)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    PERFORMANCE IMPROVEMENT PLAN                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   WHEN TO USE:                                                              │
│   • Sustained underperformance (not one-time issues)                        │
│   • After coaching and feedback have been tried                             │
│   • Clear gap between expectations and performance                          │
│                                                                             │
│   ─────────────────────────────────────────────────────────────────────     │
│                                                                             │
│   PIP COMPONENTS                                                            │
│                                                                             │
│   1. PERFORMANCE GAPS                                                       │
│      ├── Specific description of deficiencies                               │
│      ├── Examples with dates and details                                    │
│      └── Impact on team/business                                            │
│                                                                             │
│   2. EXPECTATIONS                                                           │
│      ├── Clear, measurable performance standards                            │
│      ├── Behaviors required                                                 │
│      └── What "success" looks like                                          │
│                                                                             │
│   3. SUPPORT PROVIDED                                                       │
│      ├── Training or coaching                                               │
│      ├── Resources available                                                │
```

### Conducting Difficult Conversations

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    DIFFICULT CONVERSATION FRAMEWORK                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   PREPARE                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • Gather specific examples and data                               │   │
│   │   • Anticipate employee's perspective                               │   │
│   │   • Plan key points (don't script word-for-word)                    │   │
│   │   • Choose private, appropriate setting                             │   │
│   │   • Consult HR if needed                                            │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   OPEN                                                                      │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • State purpose clearly and directly                              │   │
│   │   • Don't soften with excessive small talk                          │   │
│   │   • "I need to discuss some concerns about..."                      │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   DESCRIBE                                                                  │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • Share specific observations (SBI model)                         │   │
│   │   • Stick to facts, not interpretations                             │   │
│   │   • Explain impact on team/business                                 │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   LISTEN                                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • Ask for their perspective                                       │   │
│   │   • Listen without defending or arguing                             │   │
│   │   • Acknowledge their feelings                                      │   │
│   │   • Ask clarifying questions                                        │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   PROBLEM-SOLVE                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • Focus on future actions, not past blame                         │   │
│   │   • Set clear expectations going forward                            │   │
│   │   • Agree on support and resources                                  │   │
│   │   • Establish follow-up timeline                                    │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   CLOSE                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   • Summarize agreements                                            │   │
│   │   • Confirm next steps                                              │   │
│   │   • Express confidence (if genuine)                                 │   │
│   │   • Document the conversation                                       │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Modern Performance Management Trends

### Evolution of Practices

| Traditional | Modern Approach |
|-------------|-----------------|
| Annual reviews | Continuous feedback |
| Ratings focus | Development focus |
| Manager-driven | Employee-owned |
| Backward-looking | Forward-looking |
| Secret calibration | Transparent process |
| Individual goals | Team/company OKRs |
| Forced ranking | No/fewer ratings |

### Companies Eliminating Ratings

Several major companies have moved away from traditional ratings:

- **Adobe**: Replaced with "Check-in" conversations
- **Microsoft**: Eliminated stack ranking, moved to growth mindset
- **GE**: Moved from forced ranking to continuous feedback
- **Deloitte**: Replaced with frequent check-ins
- **Accenture**: Eliminated rankings for 90% of workforce

## Practice Exercise: Performance System Design

### Scenario

You are designing a performance management system for a 500-person software company that currently has no formal system. The CEO wants something "modern and agile" that supports the company's culture of innovation and collaboration.

**Company Context:**
- Fast-paced, project-based work
- 70% engineers, 30% business roles
- Average age: 32
- Current turnover: 25% (concerning)
- Feedback: Employees say they don't know where they stand

### Design Requirements

Create a performance management system that addresses:

1. **Goal Setting**
   - What framework will you use (SMART, OKRs, other)?
   - How frequently will goals be set and reviewed?
   - How will individual goals connect to company goals?

2. **Feedback Mechanisms**
   - What feedback practices will you implement?
   - How will you ensure feedback happens regularly?
   - What tools or technology will support feedback?

3. **Formal Reviews**
   - Will you have ratings? If so, what scale?
   - How often will formal reviews occur?
   - What will the calibration process look like?

4. **Development Integration**
   - How will performance connect to development planning?
   - How will you identify high performers vs. underperformers?
   - What actions follow from performance assessments?

5. **Implementation Plan**
   - How will you roll this out?
   - What training is needed?
   - How will you measure success of the new system?

## Key Takeaways

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    PERFORMANCE MANAGEMENT: KEY INSIGHTS                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. GOALS DRIVE ALIGNMENT. Clear, cascaded goals connect individual        │
│     effort to organizational strategy. OKRs enable transparency and        │
│     stretch thinking.                                                       │
│                                                                             │
│  2. FEEDBACK SHOULD BE CONTINUOUS. Annual reviews are insufficient.        │
│     Build cultures of ongoing feedback through 1:1s and in-the-moment      │
│     conversations.                                                          │
│                                                                             │
│  3. WHAT + HOW BOTH MATTER. Results alone are insufficient. How people     │
│     achieve results (behaviors, values) must also be assessed.             │
│                                                                             │
│  4. FAIRNESS REQUIRES CALIBRATION. Left uncalibrated, manager ratings      │
│     vary widely. Calibration ensures consistency across teams.             │
│                                                                             │
│  5. PERFORMANCE ISSUES REQUIRE ACTION. Address underperformance early      │
│     with clear expectations and support. Document everything.              │
│                                                                             │
│  6. LINK TO DEVELOPMENT. Performance management should primarily           │
│     serve growth, not just evaluation. Connect to career development.      │

```
