## Characteristics of Complex Problems

### Simple vs. Complicated vs. Complex

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    PROBLEM TYPE COMPARISON                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   SIMPLE (Following a Recipe)                                           │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Clear cause and effect                                        │  │
│   │ • Standard solution applies                                     │  │
│   │ • Expertise not required                                        │  │
│   │ • High repeatability                                            │  │
│   │                                                                 │  │
│   │ Example: Processing an expense report                           │  │
│   │ Approach: Create rules and processes                            │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   COMPLICATED (Sending a Rocket to the Moon)                            │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Cause and effect knowable with expertise                      │  │
│   │ • Multiple right answers possible                               │  │
│   │ • Expertise and analysis required                               │  │
│   │ • High repeatability once solved                                │  │
│   │                                                                 │  │
│   │ Example: Building a new ERP system                              │  │
│   │ Approach: Bring in experts, analyze thoroughly                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   COMPLEX (Raising a Child)                                             │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Cause and effect only visible in retrospect                   │  │
│   │ • Solution emerges through experimentation                      │  │
│   │ • Expertise helps but doesn't guarantee outcome                 │  │
│   │ • Every situation is unique                                     │  │
│   │                                                                 │  │
│   │ Example: Transforming organizational culture                    │  │
│   │ Approach: Probe, sense, respond, iterate                        │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   WICKED (Homelessness)                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • No definitive formulation of the problem                      │  │
│   │ • No stopping rule (never fully "solved")                       │  │
│   │ • Solutions are not true/false, but better/worse                │  │
│   │ • Every attempt changes the problem                             │  │
│   │ • Stakeholders disagree on what success looks like              │  │
│   │                                                                 │  │
│   │ Example: Balancing short-term profit and long-term investment   │  │
│   │ Approach: Continuous adaptive leadership                        │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Why Complex Problems Are Different

| Characteristic | Complicated | Complex |
|---------------|-------------|---------|
| **Cause-effect** | Knowable through analysis | Only visible in retrospect |
| **Solution** | Can be designed | Must emerge |
| **Experts** | Know the answer | Provide perspective |
| **Predictability** | High if analyzed well | Low inherently |
| **Approach** | Analyze then act | Act (probe) then learn |
| **Failure** | Due to poor analysis | Part of learning |
| **Repeatability** | Solution can be replicated | Each situation unique |

## Wicked Problems

### Characteristics of Wicked Problems

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    WICKED PROBLEM CHARACTERISTICS                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   1. NO DEFINITIVE FORMULATION                                          │
│      • Different stakeholders define the problem differently            │
│      • The formulation of the problem IS the problem                    │
│      • Understanding the problem requires solving it                    │
│                                                                         │
│   2. NO STOPPING RULE                                                   │
│      • Never "done" - just run out of time or resources                 │
│      • Can always be improved                                           │
│      • Success is relative, not absolute                                │
│                                                                         │
│   3. SOLUTIONS ARE BETTER/WORSE, NOT RIGHT/WRONG                        │
│      • No objective measure of success                                  │
│      • Different stakeholders evaluate differently                      │
│      • Trade-offs are inherent                                          │
│                                                                         │
│   4. NO IMMEDIATE OR ULTIMATE TEST                                      │
│      • Consequences unfold over time                                    │
│      • May create new problems                                          │
│      • Can't fully test in advance                                      │
│                                                                         │
│   5. EVERY ATTEMPT COUNTS                                               │
│      • No trial runs; every action has consequences                     │
│      • Can't undo and try again                                         │
│      • Solutions change the problem                                     │
│                                                                         │
│   6. INFINITE SOLUTION POSSIBILITIES                                    │
│      • No enumerable set of options                                     │
│      • Creative solutions possible                                      │
│      • No way to prove you've considered all options                    │
│                                                                         │
│   7. ESSENTIALLY UNIQUE                                                 │
│      • Despite similarities, each is different                          │
│      • History and context matter                                       │
│      • Solutions don't transfer directly                                │
│                                                                         │
│   8. SYMPTOM OF ANOTHER PROBLEM                                         │
│      • Problems are interconnected                                      │
│      • Solving one may worsen another                                   │
│      • Level of analysis matters                                        │
│                                                                         │
│   9. MULTIPLE EXPLANATIONS                                              │
│      • Different worldviews see different causes                        │
│      • Choice of explanation shapes solution                            │
│      • No way to prove which explanation is "right"                     │
│                                                                         │
│   10. NO RIGHT TO BE WRONG                                              │
│       • Decision-makers are held accountable                            │
│       • Even with uncertainty, must act                                 │
│       • "Best available" judgment required                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### CEO Wicked Problems

| Wicked Problem | Why It's Wicked |
|----------------|-----------------|
| **Balancing stakeholder interests** | No formula; each stakeholder has different needs |
| **Digital transformation** | Constantly changing; affects everything |
| **Culture change** | Emergent, takes years, never "done" |
| **Work-life balance** | Individual needs vary; trade-offs inherent |
| **Sustainability vs. profitability** | Timeframes differ; measurement disputed |
| **Innovation vs. efficiency** | Both needed; tension inherent |
| **Centralization vs. decentralization** | Benefits and costs on both sides |

## Approaches to Complex Problems

### Adaptive Leadership

```

├─────────────────────────────────────────────────────────────────────────┤

│   │ • Counteract distractions and quick fixes                       │  │

│                                                                         │
│   5. GIVE THE WORK BACK                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Don't rescue people from adaptive work                        │  │
│   │ • Resist pressure to provide "the answer"                       │  │
│   │ • Empower those who must live with consequences                 │  │
│   │ • Build capacity through doing                                  │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│   6. PROTECT VOICES OF DISSENT                                          │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Those who see differently often see important things          │  │
│   │ • Mavericks and critics provide essential intelligence          │  │
│   │ • Create safety for unpopular views                             │  │
│   │ • Listen to the song beneath the words                          │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

│   └─────────────────────────┘    └─────────────────────────┘           │
│                                                                         │
│   SAFE-TO-FAIL EXPERIMENTS:                                             │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │ • Small scale: Limit resources at risk                          │  │
│   │ • Multiple parallel: Run several probes simultaneously          │  │
│   │ • Quick feedback: Know if working within weeks, not years       │  │
│   │ • Clear success criteria: Know what you're looking for          │  │
│   │ • Fail safe: If probe fails, damage is contained                │  │
│   │ • Learning orientation: Failure is information, not defeat      │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## Key Takeaways

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                    COMPLEX PROBLEM-SOLVING: KEY POINTS                     ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                           ║
║   1. RECOGNIZE THE TYPE OF PROBLEM                                        ║
║      • Simple, complicated, complex, chaotic need different approaches    ║
║      • Most strategic problems are complex, not complicated               ║
║      • Expertise helps less in complex domains                            ║
║                                                                           ║
║   2. THINK IN SYSTEMS                                                     ║
║      • See interconnections, not just components                          ║
║      • Look below events to patterns, structures, mental models           ║
║      • Understand feedback loops and system archetypes                    ║
║                                                                           ║
║   3. EMBRACE EMERGENCE                                                    ║
║      • Complex systems can't be fully analyzed in advance                 ║
║      • Probe-sense-respond beats plan-execute-evaluate                    ║
║      • Run safe-to-fail experiments to learn                              ║
║                                                                           ║
║   4. LEAD ADAPTIVELY                                                      ║
║      • Distinguish technical from adaptive challenges                     ║
║      • Give the work back to those who must adapt                         ║
║      • Regulate distress: enough to motivate, not paralyze                ║
║                                                                           ║
║   5. ADDRESS ROOT CAUSES                                                  ║
║      • Surface-level fixes create new problems                            ║
║      • Use 5 Whys, fishbone, or current reality trees                     ║
║      • Invest in structural change, not just symptom relief               ║
║                                                                           ║
╚═══════════════════════════════════════════════════════════════════════════╝
```
